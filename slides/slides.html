Using Kubernetes as Development Environment

Steffen Hanikel / Flixbus

Background

  • Fast growing company
  • From 2 to ~100 developers in < 5 years
  • Monolith PHP application
  • 20/80 % customer/internal software
  • 3 Java / 6 PHP teams + mobile

Current situation

  • Mismatch between production/development
  • >3 Vagrant machines in use
  • get copied around when onboarding
  • hard to integrate new services

Goals

  • Robust, reproducible, easy to use
  • Teams must be able to iterate their environments quickly and independently
  • Developers should get familiar with the prod enviroment

Solution

  • Kubernetes in a Vagrant Box
  • Works on all systems
  • Minimal requirements
  • Previous experience

Demo

Challenges

  • Adoption rate
  • Initial support
  • Still high complexity
  • Caching, credentials, file permissions, fast file sharing
  • Mismatch between containers for dev/prod

Bugs

  • Routing loop for service ips crash VBox NIC
  • Some versions of Virtualbox don't set the right MAC address for the VM
  • NFS has problems with proto=udp (MTU?)

Conlusion

  • DevOps world is moving really fast right now (minikube)
